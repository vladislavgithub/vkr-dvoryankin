\section*{ЗАКЛЮЧЕНИЕ}
\addcontentsline{toc}{section}{ЗАКЛЮЧЕНИЕ}
\thispagestyle{plain}

В рамках настоящей магистерской диссертации разработан программный
сервис EduQuiz для интерактивного взаимодействия с обучающимися,
развёртываемый в инфраструктуре Российской Федерации и применимый
к преподаванию учебных дисциплин в университете. Поставленные задачи
решены полностью.

По результатам сравнительного анализа шести распространённых
платформ интерактивного обучения (Quizlet, Kahoot!, Mentimeter,
Slido, JoyTeka, OnlineTestPad), сведённых в~табл.~1.1, показано,
что ни одно из существующих решений не сочетает одновременно
работу в режиме реального времени, расширенный набор игровых
механик, размещение серверной инфраструктуры в Российской
Федерации и наличие открытого программного интерфейса. На
основании выявленного пробела сформулированы 7~функциональных
и~6~нефункциональных требований к разрабатываемому сервису.

В качестве архитектурного решения выбрана монолитная
клиент-серверная архитектура с собственным backend-приложением
и определён технологический стек: язык Go~1.25 с фреймворком
Gin для серверной части, СУБД PostgreSQL~16, сервер кеширования
Redis~7, обратный прокси Caddy~2 с автоматическим выпуском
TLS-сертификата, кроссплатформенная платформа Flutter~3.35
для клиентского приложения с единой кодовой базой под платформы
Web, iOS, Android и macOS.

В составе сервиса разработаны модели и алгоритмы: гибридная
ролевая модель пользователей с разграничением доступа по схеме
RBAC и дополнительной проверкой принадлежности ресурса; формальная
модель интерактивной учебной сессии в виде конечного автомата
с поддержкой трёх режимов проведения; модель игровой прогрессии
с формулами начисления очков опыта и вычисления уровня;
адаптация алгоритма SuperMemo SM-2 с автоматическим формированием
оценки качества ответа из пары «корректность~--- время»; модель
оценки качества банка тестовых вопросов средствами классической
теории тестов с цветовой интерпретацией значений для преподавателя.

Реализованы четыре функциональных модуля сервиса (модуль
аутентификации, модуль преподавателя, модуль проведения
интерактивной сессии, модуль обучающегося). Сервис развёрнут
в инфраструктуре Yandex Cloud с автоматическим деплоем средствами
GitHub Actions и доступен по доменному имени eduquizz.ru.

По результатам нагрузочного тестирования собственным программным
инструментом установлено, что разработанная архитектура в одиночной
серверной конфигурации устойчиво обслуживает не менее
500~одновременных обучающихся в одной интерактивной сессии при
перцентиле $p_{99}$ латентности критического пути обработки
запросов не выше 233~мс и нулевой доле ошибок класса~5xx, что
соответствует нефункциональному требованию к производительности.
По результатам функционального тестирования на демонстрационном
банке вопросов по дисциплине \textit{«Базы данных»} кафедры
автоматизированных систем управления выполнены 43~тест-кейса
по 9~основным сценариям без дефектов уровня \emph{blocker}
и \emph{critical}; дополнительно реализованы 11~модульных тестов
для критичных компонентов серверной части.

\textbf{Научная новизна} работы состоит в следующем: предложено
правило автоматического оценивания качества ответа (величины $q$
алгоритма SM-2) по корректности и латентности ответа,
обеспечивающее интеграцию интервального повторения в цикл
синхронной учебной сессии без отдельного интерфейса самооценки;
предложен архитектурный приём встраивания психометрической
аналитики банка тестовых вопросов в цикл интерактивной учебной
сессии, обеспечивающий итеративное улучшение банка в рабочем
месте преподавателя без перехода в специализированное программное
обеспечение для статистического анализа; разработана совокупность
моделей и алгоритмов, объединяющая проведение интерактивной
учебной сессии, индивидуальное интервальное повторение
и психометрический анализ в едином программном цикле, что отличает
её от существующих интерактивных обучающих платформ, реализующих
указанные функции раздельно.

Сформулированы перспективы дальнейшей научно-исследовательской
работы: полная двусторонняя интеграция с системой дистанционного
обучения университета; промышленный мониторинг и резервирование
критичных узлов; реализация адаптивного режима тестирования
на основе теории отклика на пункт; разработка модели оценки
валидности атрибуции учебного ответа в условиях распространения
общедоступных больших языковых моделей.
